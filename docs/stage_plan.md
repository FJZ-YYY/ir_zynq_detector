# Stage Plan

## 这一步的目标

把项目拆成一个对 Zynq-7020 更现实的实施路线，优先先跑通，再做优化。

## 阶段 1：最小可运行目标

输入：

- 一张热红外测试图片

输出：

- 检测类别
- 置信度
- 检测框坐标

阶段 1 不追求真实模型精度，先固定接口、输入输出格式、目录组织和基本工程流。

本仓库当前已完成的就是这一步的骨架。

## 阶段 2：PC 端训练链路

建议先缩小范围到 3 类：

- `person`
- `bicycle`
- `car`

建议首版训练策略：

- 输入分辨率：`160x128` 或 `192x160`
- 模型：`SSD-Lite + MobileNetV2` 优先
- 数据集：`FLIR_ADAS_v2`
- 训练框架：`PyTorch`
- 先导出：`ONNX`
- 再做：量化、裁剪、蒸馏中的一种

为什么先不建议大模型：

- Zynq-7020 的 ARM Cortex-A9 资源有限
- 如果没有专用 DPU，复杂模型在 PS 端运行压力很大
- 小模型更容易先做出完整演示

## 阶段 3：PS-only 板端推理

目标：

- 不依赖 PL 加速
- 在 PS 端完成图片读取、预处理、推理、后处理和 UART 输出

建议做法：

- 先抽象 `preprocess -> infer -> postprocess -> print`
- 先把模型运行时封装在 `InferEngine` 接口后面
- 真实运行时后续再选型，不先把工程写死

## 阶段 4：PL 加速

优先从固定功能下手：

- resize
- normalize
- AXI DMA
- 行缓冲 / 帧缓冲
- 简单滤波

不建议一开始就做整网硬件化，这会显著放大工程复杂度，也不符合本项目当前目标。

## 阶段 5：展示与文档

逐步补齐：

- UART 输出
- 上位机解析结果
- Demo 流程文档
- 模块接口文档
- 实验记录和性能表

## 推荐的最小接口约定

### 图片输入

- 输入格式：单张图片
- 支持格式：先以 `png / jpg / bmp` 为主

### 检测结果

统一使用下面的字段：

- `class_id`
- `class_name`
- `score`
- `bbox_xyxy`

### bbox 约定

- 格式：`[x1, y1, x2, y2]`
- 坐标系：原图像素坐标
- 左上角为原点

## 当前代码状态

- `pc/tools/mock_ir_detector.py`：可运行占位检测器
- `zynq_ps/`：板端主流程骨架
- `zynq_pl/`：PL 预处理模块占位
- `hw/vivado/create_project.tcl`：Vivado 自动建工程入口
